Project 2: Python Coded Game

Design Considerations

Overview

I will be coding a Python focused game. It is about words and basically like a Hangman program. It will be scrambled each round and you can play multiple times.

End Users

The end user of this program would be people who likes words and english in general or people that want to expand their vocabulary in a fun way. The age group that this program would be appropriate for is for aged 6+.

Requirements

What should this include:

    A list of words

    Randomiser

    Different levels of words

Specifications

A game that will showcase a whole list of words.

It should be fast and won't cause any errors when the input is different.

This would be a game that any user can play without any restrictions.


Relevant Implications

Aesthetics

To maintain and make my code look good, I will be making sure to use indents as well as spaces in between code to help our code to be futureproofed. I will also make sure that the letters will be easy to read.

Usability

I will be conducting multiple tests to ensure that the user will not be able to experience any errors and I will do this by letting my peers share their feedback regarding on this python programme.

Functionality

To ensure that the programme works the way it should work, I will be making sure to remember the wants and needs of my end users which is to have a smooth operating prgramme that has the right level of difficulty.
